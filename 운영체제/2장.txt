2.1 과제

2.2  하나 이상의 시스템 콜로 변환하여 사용자 또는 명령 파일에서 명령을 읽고 실행함
     명령어 인터프리터는 실행에 제약을 받기 때문에 완전히 사용자를 위한 프로세스여서 커널에 포함되지 않음
      유저모드에서 사용 가능

2.3 새 프로세스 시작 위해 fork 시스템 호출 후 exec 시스템을 호출함
     fork는 프로세스 복제, exec는 현재 프로세스 메모리 공간에 덮어 써버림

2.4 시스템 프로그램 : 유용한 시스템 호출의 묶음
     사용자가 일반적인 문제를 해결 위해 자신의 프로그램을 따로 작성할 필요 없도록
     사용자에게 기본적인 기능 제공

2.5 변경 사항이 운영 체제의 모든 섹션을 건드리지 않고 시스템의 제한된 섹션에만
     영향을 미치기 때문에 시스템 디버깅, 수정이 쉬움
     정보는 필요한 경우에만 보관되며 정의된 제한된 영역 내에서만 접근 가능하므로
     데이터에 영향 미치는 버그는 특정 모듈 또는 계층으로 제한되어야 함
     사용자 프로그램이 어떤 서비스를 얻기 위해서는 각 계층을 통과해야 하기 때문에 이 과정에서 오버헤드 발생

2.6 프로그램 수행, 입출력 연산, 파일 조작, 통신, error detection

2.7 휴대용 PDA, 휴대폰 같은 장치는 파일 시스템이 있는 디스크를 장치에 사용할 수 없을 수도 있
     이 경우 운영체제는 펌웨어에 저장되어야 함

2.8 부트스트랩 : 전원을 켜겨나 재부팅 할 때 적재되는 프로그램
